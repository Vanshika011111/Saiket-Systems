This repository holds the project files for the Customer Churn Analysis and Prediction task completed during the SaiKet Systems Power BI Internship program.

The core challenge for any telecom company is preventing customer turnover, or **churn**. This project focuses on building an insightful Power BI dashboard to analyze *who* is leaving, *why* they're leaving, and ultimately, to help the business identify and retain at-risk customers.

---

## üéØ Project Goal

The primary objective was to move beyond just knowing the overall churn number. We aimed to develop a dashboard that provides **actionable insights** and **recommendations** by:

1.  Analyzing the current churn rate and identifying high-impact factors (e.g., contract type, tenure, services).
2.  Creating a foundational visual analysis layer that could eventually support a predictive model to flag customers before they leave.

---

## üõ†Ô∏è The Power BI Solution

The dashboard was built through a multi-step process, ensuring the data was clean and the visuals were impactful:

### 1. Data Connection & Transformation
* The raw customer churn dataset was imported into Power BI.
* The Power Query Editor was used for **data cleaning and transformation** (e.g., handling missing values, standardizing columns) to ensure the data was suitable for accurate visualization.

### 2. Dashboard Components (Key Analyses)

The resulting dashboard includes several key sections to guide business decisions:

**Overall Churn Rate:** A prominent card visualization displaying the percentage of customers who have churned.
**Customer Demographics Analysis:** Visuals (e.g., Bar Charts, Pie Charts) showing churn distribution across key demographic factors like **gender, partner status, and dependent status**.
* **Tenure Deep Dive:** Histograms and Box Plots illustrating how **customer tenure (in months)** relates to churn.This often reveals if customers leave early on or after their initial contract.
* **Churn Factor Analysis:** Detailed visuals (Stacked Bar Charts, Heatmaps) breaking down churn based on critical service factors:
    * Contract Type (Month-to-month contracts are often a major culprit!)
    * Payment Method
    * Internet Service Type
